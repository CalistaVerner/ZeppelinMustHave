# Zeppelin Must Have

> **Построй настоящий цеппелин — не просто летающую конструкцию.**

**Zeppelin Must Have** — технический аддон для **Create Aeronautics**, который добавляет полноценные системы управления воздушным судном: полётную телеметрию, масштабируемые горелки, грейдовые бойлеры, многоцилиндровые паровые двигатели, защищённую проводку редстоуна и автоматическое удержание высоты.

Мод рассчитан на игроков и серверы, которым нужны не декоративные воздушные суда, а инженерные машины с измеряемыми характеристиками, ограничениями, развитием по грейдам, диагностикой через Engineer's Goggles, обучением через Ponder и настройкой через data pack.

---

## Основные возможности

### Штурвал воздушного судна

**Airship Helm** — серверный телеметрический блок для движущихся подуровней Sable.

Он отображает:

- имя судна и постоянный UUID подуровня;
- глобальные координаты и высоту;
- курс, тангаж и крен;
- линейную, вертикальную и угловую скорость;
- массу подуровня;
- количество и ёмкость баллонов Aeronautics;
- текущий и требуемый объём подъёмного газа;
- заполнение оболочки и сырую подъёмную силу.

Create Engineer's Goggles показывают основную полётную информацию прямо на блоке. При приседании добавляются UUID, глобальные координаты, угловая скорость и возраст телеметрического снимка.

---

### Три грейда воздушных горелок

Все горелки Zeppelin Must Have расширяют нативную реализацию **Create Aeronautics Hot Air Burner**. Поиск герметичной оболочки, заполнение газом, давление, интерполяция, частицы, звук и подъёмная сила остаются ответственностью Aeronautics.

| Горелка | Назначение | Базовый профиль |
|---|---|---:|
| Airship Burner | Базовый нагрев оболочки | `1.0×` выхода газа |
| Forced-Draft Airship Burner | Ускоренный нагрев и большие оболочки | `2.25×` выхода газа |
| Industrial Airship Burner | Тяжёлые цеппелины и крупные подъёмные системы | `4.5×` выхода газа |

Горелки поддерживают:

- ванильное и NeoForge печное топливо;
- обычное топливо Create Blaze Burner;
- сверхнагретое топливо Create;
- Creative Blaze Cake для бесконечной работы;
- одновременный запас обычного и сверхнагретого тепла;
- управление тягой аналоговым редстоуном;
- несколько горелок на одной нативной оболочке Aeronautics;
- серверные профили из data pack;
- подробную диагностику через Engineer's Goggles.

Сверхнагретый запас расходуется первым. После его исчерпания горелка автоматически переходит на обычное тепло.

---

### Слоты улучшений горелки

Каждая горелка имеет три постоянных слота:

- **Thermal** — увеличивает продолжительность работы и ёмкость теплового резервуара;
- **Airflow** — увеличивает выход и радиус поиска оболочки;
- **Control** — улучшает регулирование на малом сигнале редстоуна.

Модуль устанавливается обычным использованием по горелке. Использование Create Wrench с приседанием снимает последний установленный модуль. При разрушении горелки все улучшения выпадают.

Слоты, конфликты, допустимые цели и числовые эффекты загружаются из data pack и не зашиты в классы предметов.

---

### Грейдовые бойлеры Create

Мод добавляет три грейда сосудов высокого давления, напрямую наследующих поведение **Create Fluid Tank**.

| Грейд | Активное тепло | Сверхнагретое тепло | Максимум на колонну нагревателя |
|---|---:|---:|---:|
| Copper — Grade I | 2 | 3 | 3 |
| Brass — Grade II | 3 | 5 | 5 |
| Industrial — Grade III | 5 | 8 | 8 |

Блоки одного грейда объединяются нативной системой Create и работают как единый сосуд:

- общий запас жидкости;
- нативное подключение труб;
- поддержка Create Wrench;
- поддержка Steam Whistle;
- нативный расчёт воды и уровня бойлера;
- жидкость рендерится только контроллером;
- бесшовные connected textures без сетки отдельных блоков;
- отдельная рамка, шкала и анимированная стрелка манометра для каждого грейда.

Разные грейды не могут объединяться в один контроллер. Размер, вода, эффективность, подключённые двигатели, уровень бойлера, компаратор и Stress Units рассчитываются Create.

---

### Грейдовые паровые двигатели

Каждый бойлер имеет соответствующий паровой двигатель Create с отдельным корпусом, мощностью, нагрузкой на бойлер, числом цилиндров и анимацией.

| Двигатель | Мощность | Нагрузка | Механизм |
|---|---:|---:|---|
| Copper Steam Engine | 1024 SU | 1 единица | Один цилиндр |
| Brass Compound Steam Engine | 2560 SU | 2 единицы | Два цилиндра, противофаза 180° |
| Industrial Triple-Expansion Engine | 4608 SU | 3 единицы | Три цилиндра, фазы 120° |

Двигатели сохраняют стандартную установку вала, смену направления вращения, эффективность, overstress и интеграцию с кинетической сетью Create.

Высокий грейд эффективнее использует доступную нагрузку бойлера, но не обходит ограничения по теплу, воде, размеру сосуда и эффективности.

Грейдовый двигатель должен быть установлен на грейдовый бойлер Zeppelin Must Have. Обычные Fluid Tank продолжают использовать стандартный Steam Engine из Create.

---

### Piped Redstone

**Piped Redstone** — защищённая водостойкая аналоговая проводка для воздушных судов и плотных механизмов Create.

Каждая труба имеет шесть явных портов. Соседние трубы соединяются только тогда, когда открыты оба встречных порта.

Это позволяет строить:

- соседние линии без случайного объединения;
- компактные пересечения;
- проводку внутри тесных машинных отсеков;
- подводные и waterlogged линии;
- аналоговые сети `0..15`;
- управляемые Create Wrench порты;
- продолжение прямой линии через Create Placement Assist;
- сети с разной скоростью и дальностью по грейдам.

| Грейд | Задержка | Дальность без повторителя |
|---|---:|---:|
| Copper | 4 игровых тика | 32 ребра |
| Brass | 2 игровых тика | 64 ребра |
| Resonant | 1 игровой тик | 128 рёбер |

Смешанная сеть использует самую большую задержку и самую короткую дальность среди всех входящих в неё сегментов.

#### Native Lever

**Piped Redstone Native Lever** крепится прямо на грань трубы, автоматически открывает нужный порт и подаёт сигнал только в подключённую изолированную линию. Рукоять имеет плавную анимацию и работает в waterlogged-состоянии.

#### Waterproof Repeater

**Piped Redstone Repeater** сохраняет аналоговую силу сигнала и начинает новый сегмент дальности. Обычное нажатие переключает четыре задержки, соответствующие ванильному повторителю.

### Автоматическое управление высотой

**Altitude Gauge** — функциональный направленный датчик полёта и промежуточный контроллер тяги горелок.

Он имеет четыре режима, переключаемые Create Wrench:

1. **Altitude telemetry** — преобразует глобальную высоту в аналоговый сигнал;
2. **Vertical-speed telemetry** — показывает подъём и снижение относительно нейтрального сигнала `8`;
3. **Balloon-fill telemetry** — выводит нативное заполнение оболочки Aeronautics;
4. **Altitude Hold** — добавляет коррекцию высоты и демпфирование вертикальной скорости к заднему trim-сигналу.

Типовая схема автоматического управления:

```text
Piped Redstone Native Lever
        │ базовая тяга
        ▼
Задний вход Altitude Gauge
        │ стабилизированный выход
        ▼
Piped Redstone
        ▼
Airship Burners
```

Нажмите по прибору пустой рукой с приседанием, чтобы захватить текущую высоту и включить удержание. Повторное действие отключает режим. Deadband и ограничение скорости изменения выхода уменьшают раскачку.

## Балласт, швартовка и вертикальное управление

### Балластный бак

Ballast Tank хранит до `8000 мБ` воды и предоставляет стандартный fluid capability NeoForge для вёдер, контейнеров и труб Create. Жидкость добавляет реальную динамическую массу содержащему подуровню Sable, изменяя общую массу, центр масс и инерцию судна. Компаратор показывает заполнение, а Engineer's Goggles — текущую добавленную массу.

### Швартовочная лебёдка

Mooring Winch расширяет нативный Rope Winch из Create Simulated. Кинетическое вращение выпускает и выбирает физический трос с нативными креплениями, натяжением, разрывной силой, рендером и constraints Sable. Пришвартованный цеппелин остаётся физически симулируемым, а не замораживается на месте.

### Вертикальный двигатель

Vertical Thruster — кинетический пропеллер Aeronautics для прямого подъёма и снижения. Sable прикладывает силу в реальной позиции установки каждый физический тик, поэтому смещённые двигатели создают корректный момент. Create Wrench разворачивает тягу или меняет направление установки.

## Engineer's Goggles и Ponder

Мод интегрирован в стандартные системы документации и диагностики Create.

Engineer’s Goggles отображают:

- полётную телеметрию Airship Helm;
- топливо, тягу, выход, улучшения и состояние оболочки горелки;
- профиль грейдового бойлера;
- мощность и механику парового двигателя;
- состояние датчика и контроллера Altitude Gauge.

Категория Ponder **Zeppelin Systems** содержит сцены для:

- Airship Helm;
- горелок и слотов улучшений;
- грейдовых бойлеров;
- грейдовых паровых двигателей;
- Piped Redstone;
- автоматического удержания высоты;
- управления массой через Ballast Tank;
- физической швартовки Mooring Winch;
- вертикальной тяги Vertical Thruster.

## Настройка через data pack

Большая часть числовых параметров задаётся серверными data pack и применяется через `/reload`.

Поддерживаемые каталоги:

```text
data/<namespace>/airship_burner_profiles/*.json
data/<namespace>/airship_upgrades/*.json
data/<namespace>/boiler_grade_profiles/*.json
data/<namespace>/steam_engine_grade_profiles/*.json
data/<namespace>/piped_redstone_profiles/*.json
data/<namespace>/altitude_control_profiles/*.json
data/<namespace>/ballast_tank_profiles/*.json
data/<namespace>/vertical_thruster_profiles/*.json
```

Сервер может менять выход горелок, расход топлива, ёмкость, эффекты улучшений, преобразование тепла бойлеров, мощность и геометрию двигателей, скорость труб и параметры контроллера высоты без пересборки мода.

## Обязательные зависимости

Zeppelin Must Have — аддон, а не самостоятельный мод. Все зависимости должны быть установлены на сервере и клиентах.

| Компонент | Поддерживаемая версия |
|---|---:|
| Minecraft | `1.21.1` |
| Java | `21` |
| NeoForge | `21.1.235` или совместимая более новая сборка 21.1 |
| Create | `6.0.10` — `< 6.1.0` |
| Sable | `2.0.0` — `< 3.0.0` |
| Create Simulated | `1.3.0` — `< 2.0.0` |
| Create Aeronautics | `1.3.0` — `< 2.0.0` |

Текущая версия Zeppelin Must Have: **0.10.0**.

---

## Установка

1. Установите **Java 21**.
2. Установите **NeoForge для Minecraft 1.21.1**.
3. Установите совместимые версии Create, Sable, Create Simulated и Create Aeronautics.
4. Поместите `zeppelin_must_have-0.10.0.jar` в каталог `mods`.
5. Используйте одинаковые версии мода и зависимостей на сервере и всех клиентах.

---

## Принципы совместимости

Zeppelin Must Have расширяет существующие системы через публичные API.

Мод не использует:

- Mixins;
- access transformers;
- замену блоков Create или Aeronautics;
- параллельную симуляцию оболочек;
- собственную замену газа, давления или подъёмной силы;
- замену кинетической сети Create.

Create, Sable, Create Simulated и Create Aeronautics остаются источниками истины для своих систем.

## Мультиплеер

Мод предназначен для выделенных серверов и многопользовательских миров Create Aeronautics.

Полётная телеметрия, управление, расход топлива, профили, расчёт Piped Redstone и работа бойлеров выполняются на логическом сервере. Клиент получает синхронизированное состояние для интерфейса и рендера.

Мод обязателен на обеих сторонах.
